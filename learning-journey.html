What Changed Week to Week

A 10-week climb from static pages to deployed, API-driven apps. Hardest moments marked in red, proudest win in green.

Week 01

Hello, Web

Set up my first two repos, created a personal website with HTML. Learned how the web works under the hood — DNS lookups, HTTP requests, trace routes. Discovered DevTools and how to inspect a live page. Asked AI to build my first site and kept everything it gave me.

HTML DevTools
Week 02

Semantic Structure

Learned the HTML skeleton — <!DOCTYPE html>, <head>, <body>. Started using semantic tags and heading hierarchy. Linked local and external images. Still confused about padding, @media, and containers.

Semantic HTML Images
Week 03

CSS & First Client

Linked external stylesheets, learned selectors (descendant, class, hover). Interviewed my mom for MP1 — a recipe website. Practiced CSS Diner (levels 1–17). Began to understand the difference between static and dynamic elements.

CSS MP1
Week 04

Flexbox, Forms & First Full App

Mastered flexbox vs grid. Built forms with POST. Made sites responsive. Wrote the first full draft of MP1 — categories, search bar, email subscription. Used AI to turn my sketch into a website, but had to manually type every recipe into JSON.

Flexbox Forms MP1
Week 05

JavaScript, Showcase & the Audit

Wrote my first JavaScript (todo app, js-demo). Presented MP1 at the showcase — and realized AI had filled it with recipes that weren't my mom's. I had to manually type every single recipe and redesign the whole thing. This week I also learned to audit my work: removed cookie-cutter heroes, rounded corners, empty marketing copy. Replaced generic fonts with Gelasio, sharpened the edges, wrote meaningful descriptions. This was the turning point.

"I had to go in and change the entire website to fit my mom's preferences."
JavaScript Hardest Moment
Week 06

Interactive Web Tool (MP2)

Built a workout tracker with tabs, profile saving, and macros. Used const/let, arrays, objects, arrow functions, conditionals, DOM methods. The tabs looked right but broke on interaction — fixing one thing broke another. I had to trace every function to understand what connected where. Learned about closures, callback functions, and IIFEs along the way.

"It was hard to know why this was happening and how to prevent this by making sure I didn't delete the parts of code that were connected to the buttons."
JS Deep Dive MP2 Hardest Moment
Week 07

APIs Enter the Chat

Learned fetch(), async/await, JSON. Discovered that i++ must be in the update step of for loops. Built a dog-joke hybrid page using 2 APIs on one button. Presented MP2 at the showcase and gave a lightning talk. Began to understand how data flows from an API to the DOM.

APIs fetch
Week 08

API-Powered App — Proudest Win

Built MP3 — a hotel finder with a real API. First came the invisible API key nightmare: process.env doesn't work in the browser, key was undefined, page just showed "loading hotels..." forever. Had to learn config.js, Vercel deployment, and serverless functions.

After that, everything clicked. I separated JS into 3 files sharing one global object, handled 6 edge cases (offline, missing fields, empty results, rate limits, no WiFi, incorrect data), and designed a navy-and-white theme with Gelasio font that felt genuinely mine. Room types got checkmarks instead of emojis, card sizes were consistent, and the whole thing was responsive.

"I knew it needed the change because the rooms were different sizes yet they were all the same price."
API MP3 Proudest Win
Week 09

Browser APIs & Final Project Kickoff

Explored browser APIs (text-to-speech, dark mode, localStorage). Presented MP3. Proposed my final project — a pet adoption hub. Learned why config.js breaks GitHub Pages but works on Vercel. Discovered new free APIs I could use for the final.

Browser APIs Final
Week 10

Ship It & the State Filter

Integrated the Rescue Groups API into my final project — but the first deployment failed because my home WiFi was blocking the API data. Switched to my phone hotspot and it worked instantly. Sourced real shelter data from reliable websites, fixed FAQ buttons that weren't working, and made animal images display properly. Removed shelters.html because it was too similar to contact.html.

After demo day, I added a state filter so users can browse pets by location. The original dropdown only had three states — I expanded it to all 50 US states and connected it to the shelter data on the pet-detail popups. Adjusted card image sizes and layout to two cards per row.

"I switched from my home wifi to my phone hotspot and was able to get the API data to load, and load faster."
API Final Deploy Filter
Hardest Moments

MP1 redesign, MP2 tabs, MP3 API key

Three gut-checks that taught me more than any smooth week: AI recipes that weren't mine, tabs that broke on interaction, and an API key that silently failed. Each one forced me to fully understand what the code was actually doing.

Proudest Win

MP3 — Hotel Finder

From silent API failure to a polished, edge-case-handling, responsive app with separated JS files, a professional navy-and-white theme, and a developer who finally knew what needed to change and why.
