## Iteration 1

-The API returns an array for each of the hotels and the structure has name, url, price, currency, rating, ratingLabel, reviews, stars, description, address, image, rooms, and location.
- The fields I used were hotel.name, hotel.rating, hotel.price, hotel.currency, hotel.address.full, hotel.rooms[0].features, hotel.stars, hotel.reviews, hotel.image, and hotel.url. There is also fallback included for if these fields don't have entries.
- The hardest part of getting fetch to work was getting the API key to work and match with the code. Originally, I used process.env, but it doesn't work in the browser and the key was undefined. I didn't know the request was failing because it showed loading hotels..., but eventually never went away.
- It explained fetch() doesn't give the data immediately and await is used to get the response. I also learned that it is a chain of events and later steps rely on previous steps. Lastly, .catch catches anything that fails  and async is the same as await but written top-to-bottom instead of chained.

## Iteration 2

- The user input connects to API calls by all of the data being loaded from the API, or the fallback if it doesn't work, when the page is opened. When the user clicks on different filters, the data in the memory is filtered. The different hotels are organized into cards for each hotel and it is 3 across. There is a filter bar on top with a search box and dropdown menus. Each hotel card has a popup that displays more detailed information of the hotel and the website is responsive for mobile devices. For design decisions, I decided to separate JS files into multiple separate ones to make it easier to read and all of the data is shared through one global object. I decided to go with a navy and white theme with a Serif font because I felt that it made it more professional and clean-looking. There is also only one page with the popups.

## Iteration 3

An extension I added was the details popup that allows the user to see a more detailed description of the room types and the features/amenities of the hotel. The edge cases I handled were when there is no Wifi and the website is offline, missing fields, empty search results, and API rate-limits/missing fields/incorrect information. Something that surprised me about working with API is that APIs can have the correct information but give incorrect information and you have to look at each of the hotel cards because you could be calling the API incorrectly, causing the incorrect information to be displayed.
